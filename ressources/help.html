The syntax of the Filter Condition field

The syntax of the filter definition field defines two sections for include and exclude definitions. The include section begins with '+', the exclude section with '-'. Both sections may be omitted but the include section has to be defined first. If a part of or the whole include section is empty all directories/files are taken. If a part of or the whole exclude section is empty nothing will be exluded.

All directories to be included/excluded are listet with the ';' seperator. Behind the last named directory there may be a list of files in brackets, again seperated by ';'.

The filter algorithm will include/exclude directories and files if the specified string is part of the full pathname. The full syntax is given in the table below.

filter := [ [+]includelist ] [ -excludelist ]
includelist := [ directorylist ] [( filelist )]
excludelist := [ directorylist ] [( filelist )]
directorylist := directory [; directory [ ... ] ]
filelist := file [; file [ ... ] ]
directory := directoryname | part of directoryname | relative path
file := filename | part of filename | relative path

Examples

-/temp/will exclude everything inside any folder named 'temp'
-tempwill exclude every folder with 'temp' in its name. So the folder 'templates' will be excluded as well!
-/cache/;/Cache/(.tmp)will exclude all files inside cache folders and all files with the extension 'tmp' regardless where they are.
+(.jpg;.bmp)-/temp/will backup only JPEG and BMP images outside any 'temp' folders.
